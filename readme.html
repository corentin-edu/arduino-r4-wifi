🚀 Arduino Uno R4 WiFi - Web Dashboard Control

Projet de fin de séquence - Session 2025/2026
Développé par : Corentin PORCQ (Classe : 2TNE1)

📝 À propos du projet

Ce projet consiste en la création d'un système domotique miniature piloté par une interface web interactive. L'objectif est de transformer une carte Arduino Uno R4 WiFi en un serveur capable de gérer des sorties physiques (LEDs) et des entrées capteurs (Température/Humidité) de manière sécurisée.

🤖 Note Pédagogique

Dans le cadre de ce projet, l'utilisation de l'intelligence artificielle a été autorisée pour la programmation afin d'accompagner le développement et la compréhension du code.

Fonctionnalités clés :

  • Interface Web (Dashboard) : Une page moderne permettant d'allumer/éteindre 5 LEDs individuellement ou par groupe.
  • Modes d'animations : Intégration de cycles lumineux (Chenille, Essuie-glace, Aller-retour, Alterné).
  • Monitoring : Affichage en temps réel des données du capteur DHT22 (Température & Humidité).
  • Sécurité d'accès : Système d'autorisation manuelle via le moniteur série (Logique y/n) pour valider chaque session utilisateur.

🛠 Matériel utilisé

  • Microcontrôleur : Arduino Uno R4 WiFi
  • Capteur : DHT22 (Capteur de température et d'humidité)
  • Actionneurs : 5 LEDs + 5 Résistances de 220Ω
  • Logiciel : Arduino IDE avec bibliothèques WiFiS3 et DHT sensor library

📂 Contenu du dossier

  • arduino_code.ino : Le code source principal (C++).
  • plan-installation.pdf : Le guide complet à suivre pour l'installation et le câblage.

📖 Comment l'installer ?

⚠️ IMPORTANT : Pour mettre en service le projet, veuillez impérativement suivre les étapes détaillées dans le fichier plan-installation.pdf.

Résumé de la procédure :

  1. Téléchargement : Cliquez sur le bouton bleu en haut de cette page pour récupérer l'archive ZIP.
  2. Câblage : Connectez les 5 LEDs (Pins 9 à 13) et le capteur DHT22 (Pin 6) comme indiqué dans le schéma du PDF.
  3. Configuration : Ouvrez le code avec Arduino IDE et modifiez les variables ssid et pass avec vos identifiants WiFi.
  4. Téléversement : Envoyez le code vers la carte Arduino Uno R4 WiFi.
  5. Accès : Ouvrez le moniteur série (9600 baud), récupérez l'adresse IP et ouvrez-la dans votre navigateur web.
  6. Validation : Tapez y dans le moniteur série pour autoriser la connexion entrante.

⚙️ Détails Techniques

L'interface web est directement intégrée dans le code Arduino. Elle utilise du JavaScript pour gérer les requêtes HTTP asynchrones (Fetch API), permettant de contrôler les LEDs sans recharger la page. Le projet supporte également des raccourcis clavier pour un contrôle fluide et une réactivité optimale.

Projet réalisé dans un cadre pédagogique pour démontrer la maîtrise des serveurs web embarqués sur microcontrôleur.
